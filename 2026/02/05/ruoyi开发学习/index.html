新建业务模块

开发时保持若依原有模块结构不变，自己新加业务模块，如上图的baogang-sale，步骤如下：

1.项目根目录（例子里面是baogangProject）下新建模块，注意构建系统选择maven，在该模块pom文件下增加项目核心模块的依赖（ruoyi-framework）

2.父工程版本锁定，pom.xml(ruoyi)下添加新建模块的信息

3.ruoyi-admin文件夹下的pom文件中添加新模块的依赖

文件结构

baogang-sale 模块名

baogang-sale/src/main/resources/templates下存放html文件，里面是前端代码和javascript脚本

baogang-sale/src/main/resources/mapper是核心 SQL 层

作用：存放 MyBatis 的 XML 映射文件。
文件详情：
BgSaleIndexItemMapper.xml
BgSaleIndexMapper.xml
BgSaleItemMapper.xml
关系：这些 XML 文件一一对应下文的 Java mapper 接口。
内容：这里面写的是真正的 SQL 语句（SELECT, INSERT, UPDATE, DELETE）。
例子：Java 接口里调 selectById(1)，MyBatis 就会来这个 XML 里找对应的

baogang-sale/src/main/java/com.ruoyi是java代码层：

domain (实体层 / Model)

作用：存放 Java 类（POJO）。这里的每一个类通常对应数据库中的一张表。
内容：类中包含属性（对应表字段）以及 Getter/Setter 方法。
例子：可能有一个 BgSaleItem.java，里面包含 id, name, price 等属性。

mapper (数据访问层 / DAO)

作用：存放 Java 接口 (Interface)。这是代码与数据库交互的桥梁。
内容：定义了操作数据库的方法名，例如 selectSaleList(), insertSaleItem()。
注意：这里通常只有接口定义，没有具体的实现代码。具体的 SQL 语句写在 resources/mapper 下的 XML 文件里。

service (业务逻辑层)

作用：存放业务处理逻辑。
结构：通常包含一个接口（例如 ISaleService）和一个 impl 实现类包。
内容：这里是处理“怎么卖”的地方。比如：计算总价、校验库存是否充足、判断用户有没有权限购买等逻辑都在这里写。Controller 接收到请求后，会调用这里的方法。

controller (控制层)

作用：前端请求的入口（服务员）。
内容：定义 URL 接口（API）。比如前端发送一个 /sale/list 的请求，就是由这里的类来接收。
流程：它接收请求 -> 解析参数 -> 调用 service 层处理业务 -> 拿到结果 -> 返回给前端（JSON 数据）

总结流程

首先用户在网页中选中了一种或几种产品大类，然后$(‘#firstLevel’).on(‘change’, function (e)这个js检测到id为firstLevel的元素的内容有变化，就触发这个函数里面的操作，得到了用户选中的大类的id拼接的字符串（逗号隔开）。

然后ajax发送网络请求到url：ctx+”baogang-sale/item/listByParentIds”,这个网络请求由controller接受，listByParentIds这个方法在BgSaleItemController.java文件里

找到这个方法，这个方法调用了service文件夹下bgSaleItemService.java文件里的selectBgSaleItemById接口，但它不管怎么实现，具体实现在impl文件夹的BgSaleItemServiceImpl.java文件里

找到bgSaleItemMapper.java文件下的selectBgSaleItemById，这个文件也只提供接口，真正实现在resources文件夹的mapper文件夹的xml文件里，xml文件里是一些sql语句，把查询结果返回给了前端，前端js脚本只取了html，贴到了前端网页上，具体返回的响应看前面。
